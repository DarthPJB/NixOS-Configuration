🏗️ Architecture Overview

This repository implements a declarative infrastructure management system using NixOS flakes. The architecture follows a hub-and-spoke model with WireGuard VPN as the central communication layer.

  • → Main entry point
  • → Defines mkX86_64/mkAarch64 functions
  • → Machine configurations
  • → Deployment apps (nixinate)
  • → Build/deploy-all utilities

  • → 19 machine configurations
  • → Each with default.nix + hw-config
  • → Imports environments & services
  • → VPN configuration via enable-wg.nix
  • → Hardware-specific settings

environments/

  • → 26 environment modules
  • → Software stacks (code, browsers, etc.)
  • → Desktop environments (i3, sway, etc.)
  • → Development tools
  • → System utilities

modules/enable-wg.nix

  • → Primary VPN configuration module
  • → WireGuard interface setup
  • → SSH port configuration (1108)
  • → Secret key management
  • → Peer configuration

lib/

  • → 8 shared utility functions
  • → wg_peers.nix: VPN peer generator
  • → mkNftables.nix: NAT rule generator
  • → mkProxyPass.nix: Nginx config
  • → Reusable across machines

secrets/

  • → Encrypted via secrix
  • → WireGuard private keys
  • → SSH host keys
  • → Service credentials
  • → Public keys (safe to commit)

🚀 Deployment Architecture

The deployment system uses nixinate for remote deployment via WireGuard VPN.

1

Local Build & Validation

Build and validate configuration locally before deployment.

nixos-rebuild build --flake .#hostname
2

Test Deployment

Deploy to target machine in test mode (non-permanent).

nix run .#hostname
3

Permanent Deployment

Apply configuration permanently to target machine.

nix run .#hostname -- switch
4

Batch Deployment

Deploy to all configured hosts (whitelisted).

nix run .#deploy-all
