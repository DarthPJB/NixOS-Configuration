🔄 CI/CD Overview

The CI/CD pipeline for this NixOS configuration repository is unique because it generates GitHub Actions workflow configuration directly from Nix evaluation. This ensures that CI configuration is always in sync with actual build requirements.

✅ Key Benefits

  • Configuration as Code: CI pipeline defined in Nix, version controlled
  • Reproducibility: Same Nix evaluation produces same CI configuration
  • Consistency: CI tests exactly what you build locally
  • Automation: Workflow generation from machine definitions
  • Validation: CI configuration can be tested before deployment

# Step 1: Generate CI workflow nix run .#generate-ci-workflow # Step 2: Validate workflow nix run .#validate-ci-workflow # Step 3: Check generated file cat .github/workflows/ci.yml # Step 4: Commit to repository git add .github/workflows/ci.yml git commit -m "ci: add GitHub Actions workflow" git push

⚠️ Important Notes

The generated workflow file should be committed to the repository. This ensures that GitHub Actions can run the workflow without requiring Nix evaluation on the runner.

📋 CI Jobs

Detailed description of CI jobs and their purposes.

🔍 Validation Job

  • → Runs on all pushes and PRs
  • → Code formatting check
  • → Flake validation
  • → Dead code detection
  • → Must pass before builds

🏗️ Build x86 Job

  • → Builds 12 x86_64 configurations
  • → Parallel execution
  • → Artifact upload
  • → Build time: ~10-15 minutes
  • → Cache enabled

📱 Build ARM Job

  • → Builds 4 ARM configurations
  • → Raspberry Pi images
  • → Cross-compilation
  • → Build time: ~5-10 minutes
  • → SD card image generation

🔒 Security Job

  • → Secret detection
  • → Configuration validation
  • → Security best practices
  • → Runs on all changes
  • → Must pass before merge

🚀 Deploy Job

  • → Manual trigger only
  • → Requires workflow_dispatch
  • → Machine selection
  • → Action selection (build/test/deploy)
  • → Deployment safeguards

✅ Job Dependencies

✓

✓

✓

🚀 Deployment

Deployment procedures and safeguards.

⚠️ Deployment Requirements

Deployment requires manual trigger via GitHub Actions UI. Automatic deployment is disabled for security. All deployments go through WireGuard VPN and require proper secret access.

🔒 Deployment Security

  • → Manual trigger required
  • → Environment protection
  • → VPN access required
  • → Secret decryption needed
  • → Audit trail maintained

📋 Deployment Process

  • → Select machine from list
  • → Choose action (build/test/deploy)
  • → Run validation jobs
  • → Build configuration
  • → Execute deployment

🔄 Rollback Procedures
